Why convert Markdown to PDF instead of just sharing the .md file?

Markdown is excellent for writing — it's fast, distraction-free, and version control friendly — but raw .md files render as plain, unstyled text for most recipients unless they're using a Markdown-aware editor. A converted PDF or HTML file, on the other hand, presents your headings, lists, code blocks, and emphasis exactly as intended, which matters when you're sending documentation, proposals, or notes to someone who won't otherwise render Markdown correctly.

Keeping the conversion client-side also matters for anyone documenting sensitive internal notes, draft content, or unpublished writing — there's no reason a formatting tool needs to see or store the actual content of what you're converting, so this tool processes everything locally and never transmits your document anywhere.

Does this support code syntax highlighting?

Code blocks are rendered in a monospace font with a distinct background for readability, though full language-specific syntax highlighting isn't included in this version.

Can I use this for long documents like full reports?

Yes — the PDF export automatically paginates content across multiple pages as needed.
